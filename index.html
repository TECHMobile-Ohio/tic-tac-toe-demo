Understanding the Game Functions

The game works by changing the contents of a 9-item array (like a list) called gameBoard. The functions below read or write to this array.


Function Purpose Hint Location
initGame() Sets the starting state: clears the board, sets the player to 'X', and resets the text. Look here for setup errors.
switchPlayer() Toggles currentPlayer and updates the status text. Check if the turn is changing correctly.
checkGameEndAgainstCombos() Win/Draw Logic: Checks the current board state against the list of 8 winning lines. Check here for missing rules.
handleCellClick(e) The main control flow. Updates data, checks for win, and changes turns. Check here for timing/order errors.

*(The checkGameEndByCounting() function is included for advanced users only.)*

Debugging Challenges (Logic Breaks)

Note: Be sure to uncomment (remove "//") the problem solved flags in the code to mark that you have completed that specific problem.

  • Problem 1: Wrong Space Insertions

    A player can overwrite their opponents moves, or even keep moving after the game.

    Hint: There is code in handleCellClick() that prevents this from happening but has been commented out, rendering it useless. Just remove the "//" to add that functionality back in.

  • Problem 2: The Missing Win

    When you get three marks in the rightmost vertical column (indices 2, 5, 8), the game does not stop and continues to the next player. The winner is never declared!

    Hint: The list of rules for winning is incomplete. Check the winningCombos array near the top of the script and add the missing line.

  • Problem 3: No Ties Allowed

    If the board fills up and no one has won, the game doesn't end. Players can keep clicking full squares, and the status text never changes to "It's a tie!".

    Hint: You must explicitly check if the board is full. The logic is commented out in checkGameEndAgainstCombos(). Uncomment the Tie condition logic by removing the "//" marks.

  • Challenge Problem 4: Hard Mode Break (For Advanced Users)

    If you switch the game to use the checkGameEndByCounting() function, the game logic immediately breaks and misses wins in the second and third rows (indices 3–8). This is because the row counter resets incorrectly.

    Hint: The row counter needs to reset only when the index is divisible by 3 (the row length). Fix by adding the condition in the checkGameEndByCounting() loop.
