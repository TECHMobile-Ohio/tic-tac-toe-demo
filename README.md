# 🛠️ The Tic-Tac-Toe Debugging Challenge

Welcome to the **Tic-Tac-Toe Debugging Challenge**! 

At first glance, this looks like a classic game of Tic-Tac-Toe. But there's a catch: **the game is intentionally broken.** This repository is an interactive coding exercise designed to help you jump into a real codebase, hunt down syntax errors, and understand how HTML, CSS, and JavaScript team up to build web applications. 

Your mission, should you choose to accept it, is to find the bugs, fix the code, and deploy your working game!

---

## 🎯 Educational Objectives

By completing this challenge, you will practice:
* **Reading & Navigating Code:** Learning how to explore existing files (`index.html`, `style.css`, `script.js`).
* **Debugging Essentials:** Using your browser's Developer Tools (F12) to inspect elements and read console errors.
* **Core HTML Concepts:** Recognizing proper tag nesting, attributes, and structural syntax.

---

## 🚀 How to Take the Challenge

### Step 1: Clone the Project
1. Clone your newly forked repository to your local machine:
   ```bash
   git clone https://github.com/TECHMobile-Ohio/tic-tac-toe-demo.git

*(Note: If you don't want to clone the project, we also host this repo on GitHub Pages! [Just click here.](https://techmobile-ohio.github.io/tic-tac-toe-demo/))*

### Step 2: Spot the Bugs

Navigate into the folder and open index.html directly in any web browser.

Try to play the game. What happens? (Or rather, what doesn't happen?)

Open your code editor (like VS Code or Notepad++) and start digging into the files.

💡 Hint: Keep an eye out for missing HTML closing tags, broken links between files, typo-ridden class names, or mismatched JavaScript variables!

### Step 3: Test and Fix

Make changes in your code editor, save the files, and refresh your browser page to see if you fixed the issue.

Right-click the browser page, select Inspect, and look at the Console tab for helpful error clues.

## 🕵️‍♂️ Stuck? Need a Hint?

If you are having trouble getting the grid to display or the clicks to register, double-check these fundamental concepts:

Do all opened < section > or < div > tags have a matching closing tag?

Is the < script > tag pointing to the correct filename?

Are the JavaScript event listeners targeting the exact class names used in the HTML?

Good luck, and happy debugging! 💻✨
